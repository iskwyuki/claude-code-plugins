# claude-code-template 引き継ぎドキュメント

作成日: 2026-03-18
移行先リポジトリ: `claude-code-template`

---

## 概要

Claude Codeの機能を最大限活用するためのリポジトリテンプレート。
今後開発する各アプリのリポジトリ作成時にこのテンプレートをベースとして使用する。

---

## ユーザー要件（原文）

> 今後色んなアプリを開発する予定で、その度にリポジトリを作成すると思いますが、claude codeの機能を最大限活用するためのテンプレートみたいなものを作りたいと思っています。ベストプラクティスに則りながら、生成品質を高めるためのベースのCLAUDE.mdもそうですし、コミットやissue管理などは頻繁に行うのでskillsコマンド的に用意しておいたり、Agent Teams機能なども活用して生産性を高められるようなものを作りたいです。Agent teamsで言えば、ブラウザで情報を検索するチームと、コードベースを参照して現状の実装を確認するチームと、それらの情報を取りまとめて修正プランを立てるチームと、そのプランを実装するチームと、実装内容や修正プランをレビューするチームなど、今思いつくだけでもこれだけ機能を分けることができます。このような役割分担をAgent Teams機能を使って実現するのかClaude Codeのベストプラクティスなのかは分かりませんが、そのような仕組みを取り入れて、開発効率を最大限上げるためのリポジトリテンプレートを作成したいです。

---

## テンプレートに含めるもの

### 1. ベースCLAUDE.md

生成品質を高めるためのルール・構成をまとめたCLAUDE.md。
以下を含む:

- プロジェクト概要セクション（テンプレート利用時に記入）
- コーディング規約・スタイルガイド
- Git運用ルール
- コミュニケーション・提案スタイル
- セキュリティ方針
- 情報の保存先ガイド（skills / rules / docs の使い分け）

### 2. 頻出操作のSkills

`skills` として定義し、スラッシュコマンドで即実行できるようにする。

| スキル名 | 用途 |
|---------|------|
| commit | コミットメッセージ生成＋コミット実行 |
| issue | GitHub Issue の作成・更新・クローズ |
| pr | Pull Request の作成（テンプレート付き） |
| review | コード変更のレビュー |
| test | テスト実行＋結果解析 |

※ プロジェクト固有のスキルは各リポジトリで追加する想定。

### 3. Agent Teams構成

`.claude/agents/` にエージェント定義ファイルを配置。

#### 設計方針

- Agent Teamsは「判断を伴う複雑なタスク」に使う
- 「決まった手順の操作」（commit, issue管理等）はskillsで対応
- エージェント同士が自動チェーンするわけではなく、メインのClaudeが適切なエージェントを呼び出す形（＝「チーム」というより「専門家を呼べる体制」）
- 実装自体はメインのClaudeが行うのが現状最も効率的

#### エージェント定義（案）

| エージェント名 | 役割 | 主な使用ツール |
|--------------|------|--------------|
| researcher | Web検索で技術調査・ベストプラクティス収集 | WebSearch, WebFetch |
| codebase-analyst | 既存コード分析・影響範囲調査 | Read, Grep, Glob |
| planner | 調査結果をもとに実装計画を立案 | Read, Grep, Glob |
| reviewer | コード品質・セキュリティ・設計のレビュー | Read, Grep, Glob |

#### ユーザーが当初構想していた役割分担との対応

| ユーザー構想 | テンプレートでの実現方法 |
|------------|----------------------|
| ブラウザで情報を検索するチーム | → `researcher` エージェント |
| コードベースを参照して現状の実装を確認するチーム | → `codebase-analyst` エージェント |
| 情報を取りまとめて修正プランを立てるチーム | → `planner` エージェント |
| プランを実装するチーム | → メインのClaude（直接実装が最も効率的） |
| 実装内容や修正プランをレビューするチーム | → `reviewer` エージェント |

### 4. Rules（条件付きルール）

`.claude/rules/` に配置。特定ファイル編集時に自動読み込みされるルール。

- セキュリティ関連ファイル編集時の注意喚起
- テストファイル編集時のテスト実行リマインド
- 設定ファイル編集時の影響範囲確認

---

## 作業手順

1. GitHubに `claude-code-template` リポジトリを作成（テンプレートリポジトリとして設定）
2. Claude Codeのベストプラクティスを調査（公式ドキュメント・最新情報）
3. CLAUDE.mdテンプレートを作成
4. Skills定義を作成
5. Agent Teams定義を作成
6. Rules定義を作成
7. READMEに使い方を記載
8. 実際のプロジェクトで試用してフィードバック反映

---

## 注意事項

- Claude Codeの機能は頻繁にアップデートされるため、作成時点の最新ベストプラクティスを調査すること
- Agent Teamsの仕様・制約は作業開始時に公式ドキュメントで再確認すること
- テンプレートは汎用的に保ち、プロジェクト固有の設定は各リポジトリで追加する方針
